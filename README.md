# DESI Foundation Model (`desifm`)

**From-scratch** unimodal foundation model for DESI spectra + redshift (PHYS303 2026).

This codebase was rewritten cleanly for the final project. It is **not** a copy of the earlier `FoundationModel/` tree.

## Design

- **Spectrum tokenizer (phase 5 default)**: official [Polymathic AION](https://github.com/PolymathicAI/AION) DESI codec (`tok_spectrum_desi`, 273 tokens). Legacy in-repo `SpectrumCodec` via `--spectrum-tokenizer desifm`.
- **Redshift codec**: empirical CDF → Gaussian → uniform bins (256 levels)
- **Transformer**: encoder-decoder with modality embeddings
- **Approach A**: encoder sees `z`; auxiliary MLP on pooled encoder for joint redshift learning
- **Approach B**: encoder is spectrum-only; decoder always uses `REDMASK` at the redshift slot

## Install (local: venv only)

Use a **repo-local** virtualenv. Do not `pip install` this project into your system Python; that environment is shared with other tools and cannot be cleaned up “for this repo only” without manual `pip uninstall` decisions on your side.

```bash
bash scripts/bootstrap_venv.sh
```

Then use `.venv/bin/python` (or `source .venv/bin/activate`). Shortcuts:

```bash
make bootstrap   # same as the script above
make test        # pytest inside .venv
make smoke       # AION tokenizer + transformer smoke (requires .venv + [aion])
make test-aion   # pytest -m aion only
make download    # DR1 sample download script
```

In VS Code / Cursor, use the interpreter **`./.venv/bin/python`** (`.vscode/settings.json` points there).

## Local smoke (no FITS)

```bash
make smoke
```

Uses `--synthetic` random spectra; checkpoints land in `checkpoints/smoke_local/`.

## Phase notebooks (codec)

- `notebooks/02_phase_codec.ipynb` — **AION** tokenizer: grid resample (8704), encode batch, code histograms and sequences (`HF_TOKEN` required).
- `notebooks/03_phase_codec_eval.ipynb` — **AION** code diversity on local DR1 (heatmap, per-spectrum unique codes, flux overlay).

Sources for those two notebooks are generated by `python scripts/render_codec_notebooks.py` (edit that script, then re-run to refresh the `.ipynb` files). Phase 3 also needs `pyyaml` (`pip install -e ".[dev]"`); `matplotlib` is a core package dependency.

## Local DR1 sample (real FITS, laptop)

`notebooks/01_phase_data.ipynb` and `notebooks/03_phase_codec_eval.ipynb` download **one** iron healpix tile by default (coadd + redrock) from the [DESI public portal](https://data.desi.lbl.gov/public/) in the same survey/program order as NERSC `dr1_1k_scratch.jsonl` (first tile on the portal; typically `main/bright`). FITS go under `data/dr1_public/`; manifest `data/manifests/train_eval_dr1.jsonl` (gitignored data). CLI: `python scripts/download_dr1_local.py --training-order`.

```bash
make download
```

## AION token cache (faster training)

Encode each spectrum once on GPU, then train without running AION in the loop:

```bash
# Multi-GPU (recommended on NERSC)
python -m torch.distributed.run --nproc_per_node=4 scripts/cache_aion_tokens.py \
  --manifest data/manifests/train_eval_dr1.jsonl \
  --token-cache checkpoints/token_cache/dr1_local \
  --batch-size 16 --log-every 256

# Single GPU
.venv/bin/python scripts/cache_aion_tokens.py \
  --manifest data/manifests/train_eval_dr1.jsonl \
  --token-cache checkpoints/token_cache/dr1_local \
  --batch-size 16 --log-every 64

.venv/bin/python scripts/train_model.py \
  --manifest data/manifests/train_eval_dr1.jsonl \
  --spectrum-tokenizer aion-cached \
  --token-cache checkpoints/token_cache/dr1_local \
  --approach b --run-name smoke_cached --steps 100 --batch-size 4
```

## NERSC (interactive, SCRATCH-only)

See [docs/NERSC_INTERACTIVE.md](docs/NERSC_INTERACTIVE.md).
